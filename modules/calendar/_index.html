1.1.1. Default¶

1.1.1.1. Description¶

This module has no default command and returns an empty JSON array.

1.1.2. Academic¶

1.1.2.1. Description¶

This command returns an array of all the known academic-related events (semester start date, semester end date, etc). This command does not have any additional parameters.

1.1.2.2. Response Schema¶

{ "type" : "array",
  "items" : {
    "type" : "object",
    "properties" : {
      "end" : { "type" : "number" },
      "id" : { "type" : "number" },
      "start" : { "type" : "number" },
      "title" : { "type" : "string" }}}}

1.1.3. Categories¶

1.1.3.1. Description¶

Returns a full listing of all the currently available categories. Each category had a unique ID and may have zero or more subcategories. The ‘catid’ key in the response objects is used in the Category command to retreive a filtered list of events for a certain time period.

1.1.3.2. Parameters¶

Name Type Verb Required Notes
type string GET No Defaults to ‘Events’. Other valid values can be found by calling the ExtraTopLevels command.

1.1.3.3. Response Schema¶

{ "type" : "array",
  "items" : {
    "id"  : "schema.calendar.CategoryInfo"
    "type" : "object",
    "properties" : {
      "name" : { "type" : "string" },
      "catid" : { "type" : "string" },
      "subcategories" : {
        "type" : "array",
        "items" : { "$ref" : "schema.calendar.CategoryInfo" }}}}}

1.1.4. Category¶

1.1.4.1. Description¶

This command returns an array of the events for a specified category. An optional start and end timestamp may be specified. Behavior is undefined if end occurs before start or vice-versa

1.1.4.2. Parameters¶

Name Type Verb Required Notes
id integer GET Yes The id of the category you wish to query. The list of valid category IDs is available from the Categories command.
start integer GET No This parameter is a UNIX timestamp representing the start of a search range for events. Any events starting before this date will be ignored when building the response. If this date occurs later than the end parameter, the behavior will be undefined.
end integer GET No This parameter is a UNIX timestamp representing the end of a search range for events. Any events starting after this date will be ignored when building the response. If this date occurs earlier than the start parameter, the behavior will be undefined.
type string GET No Defaults to ‘Events’. Other valid values can be found by calling the ExtraTopLevels command.

1.1.4.3. Response Schema¶

{ "type" : "array",
  "items" : { "$ref" : "calendar.Event" }}

1.1.4.3.1. Type References¶

  • calendar.Event

1.1.5. Day¶

1.1.5.1. Description¶

This command responds with an array of objects representing events for the day represented by the time parameter or, if it is not provided, the current date is used. The event key in the response objects is the effective ID for the event and is needed when querying for additional information with the Detail command.

1.1.5.2. Parameters¶

Name Type Verb Required Notes
time integer GET No Sets which day’s event should be queried. The time parameter should be a UNIX timestamp and, if the parameter is missing, the current date will be used.
type string GET No Defaults to ‘Events’. Other valid values can be found by calling the ExtraTopLevels command.

1.1.5.3. Response Schema¶

{ "type" : "array",
  "items" : { "$ref" : "calendar.Event" }}

1.1.5.3.1. Type References¶

  • calendar.Event

1.1.7. ExtraTopLevels¶

1.1.7.1. Description¶

This command returns an list of additional default event names that can be queried using the Day command.

1.1.7.2. Parameters¶

Name Type Verb Required Notes
version integer GET No Defaults to a value of 1 if unspecified. Current valid values are 1 or 2. If version 2 is specified, the returned objects will have the hasCategories key in the returned JSON objects.

1.1.7.3. Response Schema¶

{ "type" : "array",
  "items" : {
    "type" : "object",
    "properties" : {
      "longName" : { "type" : "string" },
      "shortName" : { "type" : "string" },
      "type" : { "type" : "string" },
      "hasCategories" : {
        "type" : "boolean",
        "required" : false }}}}

1.1.7.4. Example Request¶

Request:

GET /api/index.php?module=calendar&command=extraTopLevels HTTP/1.0
Host: m.mit.edu
Accept: application/json

Response:

HTTP/1.1 200 OK
Connection: close
Content-Type: application/json; charset=UTF-8

[{
  "longName":"Today's Exhibits",
  "shortName":"Exhibits",
  "type":"Exhibits" }]

1.1.9. Search¶

1.1.9.1. Description¶

Searches events using the q parameter and returns any matching events. The search can span from the current day up to 7 days in the future.

1.1.9.2. Parameters¶

Name Type Verb Required Notes
q integer GET Yes The query string to use when searching the events. The string must be URL encoded.
span string GET No The number of days to search beyond the current day. This is currently has a maximum value of 7 days in the future

1.1.9.3. Response Schema¶

{ "type" : "object",
  "properties" : {
    "events" : {
      "type" : "array",
      "items" : { "$ref" : "calendar.Event" }},
    "span"  : {"type" : "number"}}

1.1.9.3.1. Type References¶

  • calendar.Event

1.1.9.4. Example Request¶

Request:

GET /api/index.php?module=calendar&command=extraTopLevels HTTP/1.0
Host: m.mit.edu
Accept: application/json

Response:

HTTP/1.1 200 OK
Connection: close
Content-Type: application/json; charset=UTF-8

[{
  "longName":"Today's Exhibits",
  "shortName":"Exhibits",
  "type":"Exhibits" }]
