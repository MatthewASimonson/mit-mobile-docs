1.4. Shuttles¶

Commands

  • Default
  • Routes (routes)
  • RouteInfo (routeInfo)
  • StopInfo (stopInfo)

1.4.1. Default¶

1.4.1.1. Description¶

This module has no default command and returns an empty JSON array.

1.4.2. Routes (routes)¶

1.4.2.1. Description¶

This command returns a list of all known routes, each stop on the route and a polyline (using WGS84 coordinates) showing the path the route will take. An important note is that the ‘next’ property in the ‘stops’ array is a UNIX timestamp and each prediction is an offset from the nearest ‘next’.

1.4.2.2. Parameters¶

Name Type Verb Required Notes
compact boolean GET No If True, the ‘stops’ key will be not be present in the response objects.

1.4.2.3. Response Schema¶

{"type" : "array",
  "items" : {
    "type" : "object",
    "properties" : {
      "gpsActive" : { "type" : "boolean" },
      "interval" : { "type" : "number" },
      "isRunning" : { "type" : "boolean" },
      "isSafeRide" : { "type" : "boolean" },
      "route_id" : { "type" : "string" },
      "summary" : { "type" : "string" },
      "title" : { "type" : "string" },

      "stops" : {
        "type" : "array",
        "items" : {
          "type" : "object",
          "properties" : {
            "id" : { "type" : "string" },
            "lat" : { "type" : "number" },
            "lon" : { "type" : "number" },
            "next" : { "type" : "number", "description" : "UNIX Timestamp" },
            "path" : {
              "type" : "array",
              "items" : {
                "type" : "object",
                "properties" : {
                  "lat" : { "type" : "number" },
                  "lon" : { "type" : "number" }}}},
            "predictions" : {
              "type" : "array",
              "items" : { "type" : "number" }},
            "title" : { "type" : "string" }}}}}}}

1.4.2.4. Android Example¶

URL url = new URL("http://m.mit.edu/api/index.php?module=shuttles&command=routes");
HttpURLConnection urlConnection = (HttpURLConnection) url.openConnection();
try {

  readStream(in);
finally {

1.4.3. RouteInfo (routeInfo)¶

1.4.3.1. Description¶

This command’s response contains details about the queried route id. The command takes an optional full parameter which will add a polyline representing the route to the response.

1.4.3.2. Parameters¶

Name Type Verb Required Notes
id integer GET Yes The id of the route you wish to query. The id can be retreived from the shuttles.Routes response.
full boolean GET No Include the polyline path for the route. Each point contains longitude and latitude values in the WGS84 spatial reference.

1.4.4. StopInfo (stopInfo)¶

1.4.4.1. Description¶

This command returns detail information about a single stop. This information includes data from all routes that use this stop and includes estimations for the shuttles next arrival time. The response also includes data indicating if the shuttle has active location assistance (the predictions may be more accurate if it does). Like the Routes (routes) and RouteInfo (routeInfo) commands, all times are represented by UNIX timestamps and the predictions are integer offsets.

1.4.4.2. Parameters¶

Name Type Verb Required Notes
id string GET Yes The id of the route you wish to query. The id can be retreived from the shuttles.Routes response.

1.4.4.3. Response Schema¶

{ "type" : "object",
  "properties" : {
    "now" : { "type" : "integer", "description" : "UNIX Timestamp" },
    "stop" : {

        "type" : "object",
